There are some super strong conditional relationships here, only 41 of the 64 possible label combinations actually occur even once knowing the values of some of the more common labels can significantly improve our ability to predict some of the rarer ones. I'm not sure what the best way to take advantage of this structure might be. There are a few obvious things we could do like for example make sure that we always mark a comment as toxic if we mark it as severe toxic.

I think there may be other interesting things to do which may be better though. For example we can choose a factorization of the total probabilities above into a bayesian network where the observation is the comment and each conditional probability is provided by a different model (or an esemble of different models).

For example because severe toxic is a pure subset of toxic we can (and should I think) factorize the probability of severe_toxic as a conditional probability,

$$ p(ST=1) = P(ST=1 | T=1)P(T=1) + p(ST=1 | T=0)p(T=0) $$

Because you never have a severe toxic comment that isn't toxic the second term drops out and we have just P(ST) = P(ST | T)P(T). Factorizing the probability this way will automatically enforce the logical connection between toxic. We can implement P(T) as any sort of model (an LSTM, or a Logistic Regression on word count vectors whatever just so long as it gives us a probability). We can train P(T) on all the comment data and then train a P(ST | T) model using just those comments which are labeled as toxic. Actually so that the pool of training data roughly matches what we would see at predict time we should probably train P(ST |T) on all the data which passes some probability threshold for our toxicity detector as well as all the actually toxic comments.

This is potentially a complicated way to approach the problem since it would require training not just one model per type of toxicity but one model for every entry in the equivalent conditional probability table for our bayes net.

Cross Category AUC ROC boost¶

In order to gauge the utility of using conditional probability models in this way I want to estimate what sort of AUC ROC scores we could get simply by predicting labels entirely based on the values of other toxicity classes. For comparison completely random label assignments would come in at a score of 0.5

In [51]:
import sklearn.metrics

cross_aucs = np.ones((6, 6))

for row_idx, conditioning_col in enumerate(train_labels.columns):
    for col_idx, target_col in enumerate(train_labels.columns):
        if conditioning_col == target_col:
            continue
        cauc = sklearn.metrics.roc_auc_score(
            train_labels[target_col].values, 
            train_labels[conditioning_col].values
        )
        cross_aucs[row_idx, col_idx] = cauc
        print("{:15s} standing in for   {:15s}  {:0.4f}".format(conditioning_col, target_col, cauc))
